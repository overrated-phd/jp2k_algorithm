§1 — Overview

What is JPEG 2000?

A wavelet-based image compression standard superseding JPEG, offering superior rate-distortion, lossless capability, and progressive transmission in a single unified codestream.

JPEG 2000, standardized in 2000 by the Joint Photographic Experts Group, replaces JPEG's 8×8 block Discrete Cosine Transform (DCT) with a full-frame Discrete Wavelet Transform (DWT). This eliminates blocking artifacts at low bitrates and enables progressive decoding, Region of Interest (ROI) coding, and unified lossless/lossy compression — features impossible with block-DCT.

The standard spans multiple parts: Part 1 (core codec), Part 2 (extensions), Part 3 (Motion JPEG 2000), Part 9 (JPIP interactive protocol), and Part 15 (HTJ2K high-throughput). Primary domains include digital cinema (DCI mandates JP2K for theatrical distribution), medical imaging (DICOM lossless archival), satellite/geospatial (Sentinel-2, GOES), and cultural heritage preservation.

Key Innovation: Embedded Coding

Unlike JPEG where each quality level requires a separate encoding pass, JP2K produces a single embedded bitstream truncatable at any point to yield the optimal reconstructed image at that rate. This is enabled by EBCOT's rate-distortion optimized layering — a concept absent from the original JPEG standard.

§3 — Mathematical Foundations

Stage-by-Stage Analysis

3.1 Tiling

The image W × H is partitioned into non-overlapping tiles of size T_w × T_h. Each tile is compressed independently, bounding memory to O(T_w · T_h). Single-tile mode (T_w = W) avoids seam artifacts.

3.2 DC Level Shift

DC Level Shiftx'(i, j) = x(i, j) − 2B−1

For bit depth B = 8, this maps [0, 255] → [−128, 127], centering the signal around zero for improved transform efficiency.

3.3 Color Transform

Irreversible Color Transform (ICT) — Lossy Mode
Forward ICT (RGB → YCbCr)Y = 0.299·R + 0.587·G + 0.114·B
Cb = −0.16875·R − 0.33126·G + 0.5·B
Cr = 0.5·R − 0.41869·G − 0.08131·B
Reversible Color Transform (RCT) — Lossless Mode
Forward RCT (integer-only)Y = ⌊(R + 2G + B) / 4⌋
Cb = B − G
Cr = R − G
Inverse RCTG = Y − ⌊(Cb + Cr) / 4⌋
R = Cr + G
B = Cb + G

The RCT uses integer division only, guaranteeing bit-exact invertibility for lossless compression.

3.4 Discrete Wavelet Transform

The DWT decomposes the image into a multi-resolution representation via separable 1-D filtering along rows then columns. Each level produces four subbands:

LL

Approximation

Low-pass in both directions. Contains coarse-scale image. Recursively decomposed at subsequent levels.

LH

Horizontal Detail

Low-pass horizontal, high-pass vertical. Captures horizontal edges.

HL

Vertical Detail

High-pass horizontal, low-pass vertical. Captures vertical edges.

HH

Diagonal Detail

High-pass both directions. Captures diagonal textures and noise.

3.5 The Lifting Scheme

JP2K implements DWT via the lifting scheme (Sweldens, 1996), factorizing the polyphase matrix into alternating predict/update steps. This yields an in-place, fully invertible algorithm with ~50% fewer operations than direct convolution.

Le Gall 5/3 — Lossless Integer Lifting
Split into even/oddsj(0) = x(2j), dj(0) = x(2j + 1)

Predict (high-pass)dj = dj(0) − ⌊(sj(0) + sj+1(0)) / 2⌋

Update (low-pass)sj = sj(0) + ⌊(dj−1 + dj + 2) / 4⌋

Floor operations ensure integer-to-integer mapping. Inverse: undo update, then undo predict.

CDF 9/7 — Lossy Real-Valued Lifting
Four lifting stepsd(1) = d(0) + α·(sj(0) + sj+1(0)) α = −1.586134342
s(1) = s(0) + β·(dj-1(1) + dj(1)) β = −0.052980118
d(2) = d(1) + γ·(sj(1) + sj+1(1)) γ = 0.882911075
s(2) = s(1) + δ·(dj-1(2) + dj(2)) δ = 0.443506852

Normalizations = K · s(2), d = (1/K) · d(2), K = 1.230174105

CDF 9/7 provides ~1 dB PSNR improvement over 5/3 at equivalent bitrates, at the cost of irreversibility.

3.7 Dead-Zone Scalar Quantization

Forward quantizationqb(i,j) = sign(ab(i,j)) · ⌊ |ab(i,j)| / Δb ⌋

The dead zone around zero has width 2Δ_b (twice normal bin width), aggressively zeroing small coefficients — the primary lossy compression mechanism.

3.8 Step Size Parameterization

Step size from header parametersΔb = 2Rb − εb · (1 + μb / 211)

Rb = nominal dynamic range of subband b
εb ∈ {0, …, 31} — exponent (5 bits)
μb ∈ {0, …, 2047} — mantissa (11 bits)

3.9 Dequantization

Reconstruction with offset δâb(i,j) = Δb · (qb(i,j) + sign(q)·δ) for q ≠ 0
âb(i,j) = 0 for q = 0

δ ∈ (0, 1) is the reconstruction offset, typically δ = 0.5 (midpoint).
Lossless Mode

Step sizes are set to Δ_b = 1 for all subbands. Combined with the integer 5/3 DWT, every coefficient passes through EBCOT without any information loss.

3.10 EBCOT Architecture

EBCOT (Taubman, 2000) operates in two tiers: Tier-1 performs context-adaptive arithmetic coding per code-block; Tier-2 performs global rate-distortion optimization across all blocks.

Code-Block Partitioning

Each subband is divided into 2n × 2m code-blocks (typically 64×64, constrained n+m ≤ 12). Independent coding enables fine-grained rate control and error resilience.

Tier-1: Three Coding Passes per Bitplane

Significance Propagation → Magnitude Refinement → Cleanup

Pass 1 (Significance Propagation): Encodes bits of not-yet-significant samples with ≥1 significant 8-connected neighbor. Context: neighbor significance state.
Pass 2 (Magnitude Refinement): Refines previously significant samples. Each bit halves the reconstruction interval width.
Pass 3 (Cleanup): Encodes all remaining samples via run-length + MQ coding.

3.11 MQ Arithmetic Coder

Context formation for significance codingκsig = f(σh, σv, σd)

σh = # significant horizontal neighbors ∈ {0,1,2}
σv = # significant vertical neighbors ∈ {0,1,2}
σd = # significant diagonal neighbors ∈ {0,1,2,3,4}

→ 9 significance contexts (LL/LH) + 9 (HL) + 1 sign + 3 refinement = 19 total

The MQ coder adapts via a 47-state finite automaton per context, achieving near-entropy efficiency.

3.12 Tier-2: PCRD Optimization

Lagrangian optimizationminimize Dtotal = Σi Di(ni)
subject to Σi Ri(ni) ≤ Rtarget

Unconstrained form: J = Σi [ Di(ni) + λ · Ri(ni) ]

i = code-block index, ni = retained coding passes
Di(n) = MSE contribution after n passes
Ri(n) = rate in bytes after n passes
λ = Lagrange multiplier (bisection search → Rtarget)

The rate-distortion slopes S = −ΔD/ΔR are sorted globally across all code-blocks. Passes are included in decreasing slope order until the rate budget is exhausted — provably optimal for any target bitrate.

3.13 Codestream Organization

Hierarchical structure: Tiles → Tile-Parts → Packets → Code-Block Contributions. Each packet aggregates data from one precinct, one resolution, one quality layer, one component.

Progression Orders

LRCP

Quality Progressive

Layer → Resolution → Component → Position. Each layer improves the entire image. Best for progressive quality refinement.

RLCP

Resolution Progressive

Resolution → Layer → Component → Position. Low-res first, then adds detail. Ideal for thumbnail → full-size.

RPCL

Spatial Progressive

Resolution → Position → Component → Layer. Enables decoding specific spatial regions per resolution level.

3.14 JPIP Interactive Protocol

JPIP window requestRequest = (fx, fy, fw, fh, roff, rsiz, layers, comps)

(fx, fy, fw, fh) = spatial ROI window
roff, rsiz = resolution range
layers = max quality layers
comps = component indices

JPIP enables browsing petapixel images without downloading the full file — the server streams only the requested packets.

3.15 JP2 Container Format

The .jp2 file uses an ISOBMFF box structure: jP (signature), ftyp (file type), jp2h (header with ihdr + colr + res), jp2c (codestream). Supports embedded ICC profiles, XML metadata, and UUID extension boxes.

§6 — Machine Learning Era

JPEG 2000 in Modern AI Pipelines

JP2K intersects ML as both competitor (learned compression) and complement (efficient features, domain-specific workflows).

1 Learned Image Compression vs. JP2K

Neural codecs (Ballé 2018, Cheng 2020, ELIC 2022) replace DWT + EBCOT with end-to-end learned components:

Neural Codec Frameworky = ga(x; θ) — analysis transform (encoder CNN/ViT)
ŷ = Q(y) — quantization (uniform + additive noise during training)
x̂ = gs(ŷ; φ) — synthesis transform (decoder CNN/ViT)

Rate-Distortion Training LossL = R(ŷ) + λ·D(x, x̂) = 𝔼[−log₂ pŷ(ŷ)] + λ·‖x − x̂‖²

pŷ = learned entropy model (hyperprior + autoregressive context model)
λ controls the rate-distortion tradeoff along the Pareto frontier

State-of-the-art neural codecs achieve 20–40% bitrate savings over JP2K at equivalent PSNR. However, they require GPU decode, lack standardization, have no progressive/ROI capability, and cannot guarantee lossless compression — JP2K remains dominant where these matter.

2 DWT as Neural Network Feature Extraction

Wavelet subbands are mathematically equivalent to fixed convolutional filter banks with stride-2 downsampling — structurally identical to a CNN's first layer:

EquivalenceLL = (hlo ⊛ x) ↓2 ≡ Conv2d(C, C, k=5, stride=2, weights=hlo)
LH = (hhi ⊛ x) ↓2 ≡ Conv2d(C, C, k=5, stride=2, weights=hhi)

Advantage over max-poolingDWT: preserves spatial structure + frequency separation (invertible)
MaxPool: discards position + high-frequency info (irreversible)

WaveCNet (Li et al., 2020) replaces all pooling with DWT → noise-robust classification. Wavelet Scattering (Mallat, 2012) cascades DWT + modulus → provably Lipschitz-continuous, translation-invariant features.

3 Video Coding for Machines (MPEG-VCM)

Compression optimized for machine consumption (detection, segmentation) rather than human viewing. JP2K's progressive structure is a natural fit:

Task-Aware Progressive DecodingRes-0 (LL only, 1/2L scale): Object detection → ~90% mAP, ~3% bandwidth
Res-1 (+LH₁,HL₁,HH₁): Improved localization
Res-2 (+LH₂,HL₂,HH₂): Fine segmentation masks
Full resolution: Human verification / archival

4 Medical Imaging & Federated Learning

JP2K lossless is mandatory in DICOM for radiological archival. For federated learning across hospitals, lossless compression eliminates codec-induced domain shift:

Domain Shift from Lossy CodecsP(x | site_A, JPEG_q85) ≠ P(x | site_B, JPEG_q92) ≠ P(x | site_C, lossless)

JP2K lossless guarantees:
P(x | any_site, JP2K_lossless) = P(x | original)
→ compression is eliminated as a confounding variable across federated sites

Whole-slide pathology images (100K×100K px) use JP2K + JPIP for tile-based access during both clinical review and ML training.

5 Neural Post-Processing of JP2K Artifacts

JP2K produces ringing (Gibbs-like oscillations) near edges instead of blocking. DNNs can be trained as post-filters:

Artifact Removal Trainingx̂enh = fθ(x̂JP2K)

L = ‖fθ(Decode(Encode(x, Δ))) − x‖₁ + λp·‖φ(fθ(·)) − φ(x)‖₂

φ = VGG perceptual features
Architectures: DnCNN, SwinIR, Restormer, HAT
Typical gain: +1.5 to +3.0 dB PSNR at 0.25 bpp

6 Satellite & Geospatial ML at Scale

Earth observation satellites (Sentinel-2, Landsat-9, GOES) store multispectral imagery in JP2K. Partial decode via JPIP reduces training I/O dramatically:

I/O SavingsFull tile: 10,980 × 10,980 × 13 bands × 16 bit ≈ 3 GB
Partial decode (256×256 crop, 3 bands, layer 2): ~150 KB
Training I/O reduction: ~20,000× per sample

Frameworks like rasterio and GDAL exploit JP2K codestream structure for efficient windowed reads at petabyte scale.

wavelet_pooling.py
import torch import torch.nn as nn import pywt class DWTPooling(nn.Module): """Drop-in replacement for MaxPool2d using DWT. Outputs 4× channels at half spatial resolution. bior4.4 ≈ CDF 9/7 used in JPEG 2000 lossy mode.""" def __init__(self, wavelet='bior4.4'): super().__init__() w = pywt.Wavelet(wavelet) self.register_buffer('lo', torch.tensor(w.dec_lo, dtype=torch.float32)) self.register_buffer('hi', torch.tensor(w.dec_hi, dtype=torch.float32)) def _filt(self, x, filt_row, filt_col): B, C, H, W = x.shape K = filt_row.shape[0]; pad = K // 2 # Row filter xp = nn.functional.pad(x, [pad,pad], mode='reflect') kr = filt_row[None,None,:,None].expand(C,1,-1,-1) r = nn.functional.conv2d(xp, kr, groups=C)[..., ::2] # Col filter rp = nn.functional.pad(r, [0,0,pad,pad], mode='reflect') kc = filt_col[None,None,None,:].expand(C,1,-1,-1) return nn.functional.conv2d(rp, kc, groups=C)[:,:,::2,:] def forward(self, x): # Returns (B, 4C, H/2, W/2) — LL, LH, HL, HH concatenated LL = self._filt(x, self.lo, self.lo) # approximation LH = self._filt(x, self.lo, self.hi) # horizontal detail HL = self._filt(x, self.hi, self.lo) # vertical detail HH = self._filt(x, self.hi, self.hi) # diagonal detail return torch.cat([LL, LH, HL, HH], dim=1) # Usage model = nn.Sequential( nn.Conv2d(3, 64, 3, padding=1), nn.ReLU(), DWTPooling('bior4.4'), # 64 → 256ch, H/2×W/2 nn.Conv2d(256, 128, 3, padding=1), nn.ReLU(), nn.AdaptiveAvgPool2d(1), nn.Flatten(), nn.Linear(128, 10), )
